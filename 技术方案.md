# 巡防航线管理模块技术方案

## 1. 高德地图API密钥需求

**需要项目方提供高德地图API密钥**。

原因：
- 当前系统在`web/index.html`中使用了占位符`YOUR_AMAP_KEY`（第11行）
- 前端代码在`Route.vue`的`initMap`函数中（第135-138行）检查了`window.AMap`是否存在，若不存在会提示用户配置高德地图key
- 高德地图API需要有效的密钥才能正常使用，否则地图无法加载

## 2. 技术方案

### 2.1 系统架构

- **前端**：Vue 3 + Element Plus + 高德地图JS API
- **后端**：Spring Boot + MyBatis Plus
- **数据库**：MySQL（根据项目结构推断）

### 2.2 核心功能实现

#### 2.2.1 前端实现

1. **地图初始化**：
   - 使用高德地图JS API初始化地图实例
   - 配置地图中心和缩放级别
   - 绑定地图点击事件，用于添加航点

2. **航点管理**：
   - 点击地图添加航点标记
   - 右键点击删除航点标记
   - 自动更新航线路径

3. **航线绘制**：
   - 根据添加的航点自动绘制航线
   - 实时更新航线路径
   - 在表单中保存航点坐标数据

4. **数据交互**：
   - 新增/编辑航线时提交表单数据
   - 查看航线详情时在地图上展示航线

#### 2.2.2 后端实现

1. **数据模型**：
   - `PatrolRoute`实体类，包含id、routeName、points、expectedDuration、flightHeight、remark等字段
   - `points`字段存储航点坐标，使用JSON格式字符串

2. **API接口**：
   - `GET /api/route/page`：分页查询航线列表
   - `GET /api/route/{id}`：根据ID获取航线详情
   - `GET /api/route/list`：获取所有航线
   - `POST /api/route`：新增航线
   - `PUT /api/route`：更新航线
   - `DELETE /api/route/{id}`：删除航线

3. **业务逻辑**：
   - 航线的增删改查操作
   - 权限控制（仅管理员可操作）
   - 操作日志记录

## 5. 操作流程

### 5.1 新增航线流程

1. **进入航线管理页面**：点击左侧菜单的"航线管理"
2. **点击"新增航线"按钮**：打开新增航线对话框
3. **填写航线基本信息**：输入航线名称、预计时长、飞行高度、备注
4. **在地图上添加航点**：点击地图任意位置添加航点，可添加多个航点
5. **调整航点**：右键点击航点可删除，可重复添加和删除操作
6. **查看航线预览**：系统会自动根据添加的航点绘制航线
7. **点击"确定"按钮**：提交表单，保存航线信息
8. **查看操作结果**：系统提示"新增成功"，并刷新航线列表

### 5.2 编辑航线流程

1. **进入航线管理页面**：点击左侧菜单的"航线管理"
2. **找到要编辑的航线**：在列表中找到目标航线
3. **点击"编辑"按钮**：打开编辑航线对话框
4. **修改航线信息**：可修改航线名称、预计时长、飞行高度、备注
5. **调整航点**：可添加新航点、删除现有航点
6. **查看航线预览**：系统会实时更新航线绘制
7. **点击"确定"按钮**：提交表单，更新航线信息
8. **查看操作结果**：系统提示"编辑成功"，并刷新航线列表

### 5.3 查看航线详情流程

1. **进入航线管理页面**：点击左侧菜单的"航线管理"
2. **找到要查看的航线**：在列表中找到目标航线
3. **点击"详情"按钮**：打开航线详情对话框
4. **查看航线信息**：查看航线的基本信息和地图上的航线展示
5. **点击"取消"按钮**：关闭详情对话框

### 5.4 删除航线流程

1. **进入航线管理页面**：点击左侧菜单的"航线管理"
2. **找到要删除的航线**：在列表中找到目标航线
3. **点击"删除"按钮**：弹出确认对话框
4. **确认删除**：点击"确定"按钮
5. **查看操作结果**：系统提示"删除成功"，并刷新航线列表

## 6. 注意事项

1. **API密钥安全**：
   - 高德地图API密钥应妥善保管，避免泄露
   - 建议在生产环境中使用服务器端代理方式调用高德地图API

2. **性能优化**：
   - 对于大量航点的航线，应考虑分批加载和渲染
   - 地图操作应添加防抖处理，避免频繁触发事件

3. **用户体验**：
   - 应添加加载状态提示，提升用户体验
   - 应提供清晰的操作指引和错误提示

4. **兼容性**：
   - 应测试不同浏览器的兼容性
   - 应考虑移动设备的适配

5. **数据安全**：
   - 后端应验证用户权限，防止未授权操作
   - 应记录操作日志，便于追溯

## 7. 总结

本技术方案基于现有的无人机巡防管控系统，实现了基于高德地图API的巡防航线可视化绘制功能。系统支持用户通过手动点击地图界面选择多个点位，并将这些点位按顺序自动连接形成完整的巡防航线。

主要功能包括：
- 地图初始化和航点管理
- 航线自动绘制和实时更新
- 航线的增删改查操作
- 权限控制和操作日志记录

通过本方案的实施，用户可以直观地在地图上规划和管理巡防航线，提高工作效率和准确性。
